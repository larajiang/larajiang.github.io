deploy-blog

基本命令：

$ hexo g #完整命令为hexo generate,用于生成静态文件
$ hexo s #完整命令为hexo server,用于启动服务器，主要用来本地预览
$ hexo d #完整命令为hexo deploy,用于将本地文件发布到github上
$ hexo n #完整命令为hexo new,用于新建一篇文章

1、安装git
sudo apt-get install git-core
2、安装node.js
2.1安装 nvm
wget -qo- https://raw.github.com/creationix/nvm/master/install.sh | sh
2.2 restart the terminal and RUN “nvm install 0.12”
3、安装Hexo
npm install -g hexo-cli
4、Once Hexo is installed, run the following commands to initialise Hexo in the target hexo


hexo init hexo          
cd hexo
npm install

5、github 新建 repository

6、配置 _config.yml

7、本地运行与调试
hexo server
网页运行在http://localhost:4000

8、部署到Github


hexo clean   #清除缓存文件（db.json) 和hexo generate 生成的文件（public）
hexo generate  #生成静态文件，存储在public 文件夹中
hexo deploy #部署你的网页

此时，在hexo根目录下会生成一个文件夹：.deploy
因为hexo版本的问题，没有出现该文件夹，出现
ERROR Deployer not found : github
运行以下命令,或者直接先运行这个命令更好。


$ npm install hexo-deployer-git —save
$ hexo g
$ hexo d

这时在hexo根目录下出现了一个.deploy_git 文件夹
然后运行
hexo generate
hexo deploy
出现 error: src refspec master does not match any.
这时因为 push 的文件为空，运行
git commit -am “message”
重新运行 hexo deploy
成功！
注意：
在浏览器中输入username.github.io就见到成果了。注意hexo deploy的部署过程：将public中文件复制到.deploy中，然后就是利用Git来push到远程GitHub仓库了。
9.problem shooting：
1、安装完成后，每次重启终端，都无法找到hexo命令“can not found command hexo”
原因：没有指定node的版本
解决方案：
a.删除 ~/.nvm ~/.npm 来uninstall nvm
b.nvm use 0.10.35
nvm alias default 0.10.35
c. 重新进行第二步
10.后续：
1、发表文章
hexo new [layout] <br>2、编写文章<br>在hexo\source_posts 中打开 **.md<br>使用Markdown<br>3、写文章后使用hexo g 生产静态文件<br> hexo s 在本地查看效果<br> hexo d 同步到github，使用larajiang.github.io 进行访问。
